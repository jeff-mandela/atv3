Como funciona a regra da cascata no CSS?

A regra da cascata define a ordem de aplicação dos estilos quando há conflitos. A prioridade é determinada por: importância (uso do !important), especificidade e ordem no código.

Especificidade no CSS

É a forma como o CSS calcula qual seletor tem mais peso:

  • Tag: Menor peso (ex: p)
  • Classe: Peso intermediário (ex: .titulo)
  • ID: Maior peso (ex: #principal)

Combinação de Seletores

Exemplos:

  • .primeiro h1: Seleciona h1 dentro de qualquer elemento com classe primeiro
  • .primeira > h1: Seleciona apenas h1 filho direto
  • .primeira h1 + h1: Seleciona o h1 que vem imediatamente após outro h1

Pseudo-classes

Servem para estilizar elementos em estados específicos:

  • :first-child → Primeiro filho
  • :last-child → Último filho
  • :nth-child(n) → Filho de número específico
  • :nth-of-type(n) → Filho de determinado tipo

Display Inline vs Block

Inline: Ocupa apenas o conteúdo (ex: span). Não aceita altura/largura.

Block: Ocupa toda a largura disponível (ex: div, p).

Elementos por padrão

  • Block: div, section, p, h1-h6
  • Inline: span, a, strong

Box Model

O modelo de caixa é composto por: content, padding, border e margin. Exemplo de shorthand:

  • 1 valor: margin: 10px;
  • 2 valores: margin: 10px 20px;
  • 3 valores: margin: 10px 20px 30px;
  • 4 valores: margin: 10px 20px 30px 40px;

Box Sizing

box-sizing: border-box; faz com que padding e border sejam incluídos no tamanho total do elemento.
